# site

Generated static files for [www.kubernetools.com](https://www.kubernetools.com/). Part of the [`kubernetools`](https://github.com/kubernetools) project — see [`kubernetools/project`](https://github.com/kubernetools/project) for the full architecture.

## Repository layout

```
site/
  index.html        # Homepage — hand-maintained, not generated
  mcp/
    index.html      # MCP server docs for end users — hand-maintained, not generated
  robots.txt        # Generated by docgen
  sitemap.xml       # Generated by docgen (latest URLs only)
  docs/
    latest/         # Generated by docgen --is-latest (physical directory, stable URLs)
    v1.36/          # Generated — one directory per supported version
    v1.35/
    …
```

`site/index.html` and `site/mcp/index.html` are the only files under `site/` that are edited by hand. Everything else is produced by [`docgen`](https://github.com/kubernetools/docgen) and must not be edited directly.

## How the site is generated

Site content is produced by the `docgen` Rust CLI, which reads the official Kubernetes OpenAPI specs from [`kubernetools/specs`](https://github.com/kubernetools/specs) and emits static HTML.

Four Kubernetes minor versions are supported at any time (N to N−3, currently v1.33–v1.36), matching the [Kubernetes community support window](https://kubernetes.io/releases/patch-releases/#support-period).

`/docs/latest/` is a physical directory generated directly by `docgen --is-latest`. Links within it use stable `/docs/latest/…` paths that do not change across Kubernetes releases, so bookmarks and search-engine indices stay valid. Version-specific paths (`/docs/v1.36/`, etc.) are generated separately and marked `noindex` by Nginx.

## Workflows

### `build-release` (manual)

Triggered from the Actions tab or via:

```bash
gh workflow run build-release.yml -f latest_version=v1.36
```

| Input | Example | Description |
|---|---|---|
| `latest_version` | `v1.36` | Version to generate as `latest`. The three previous minor versions are built automatically. |

Steps:
1. Downloads the latest `docgen` binary from its GitHub releases.
2. Removes generated content (`site/docs/latest`, `site/docs/v*`, `site/robots.txt`, `site/sitemap.xml`) while leaving `site/index.html` intact.
3. Runs `docgen generate --is-latest` for the latest version.
4. Runs `docgen generate` for the three preceding minor versions.
5. Opens a pull request with the regenerated files.

### `release` (on tag)

Triggered automatically when a tag matching `v*` is pushed. Packages `site/` as a `.tgz` archive and publishes it as a GitHub release. The [`kubernetools/deploy`](https://github.com/kubernetools/deploy) workflow picks up this archive to build the Nginx container image.

## Updating the homepage

Edit `site/index.html` directly (version list, prose, etc.) and open a PR. The `build-release` workflow does not touch this file.

## Updating the MCP docs page

Edit `site/mcp/index.html` directly and open a PR. The `build-release` workflow does not touch this file.
